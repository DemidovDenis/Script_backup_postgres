# Резервное копирование PostgreSQL – Bash скрипт

Данный скрипт создаёт сжатые gzip-дампы всех пользовательских баз данных PostgreSQL, проверяет их целостность и сохраняет в директорию `/backups`.

## Требования

- **Операционная система**: Linux (любой дистрибутив с Bash 4+)
- **PostgreSQL** (клиентские утилиты `pg_dump`, `psql`)
- Утилиты: `gzip`, `gunzip`, `df`, `date`, `mkdir`, `mv`, `rm`
- Права на запись в `/backups`
- Права на чтение всех баз данных для указанного пользователя PostgreSQL
- Настроенный способ аутентификации:
  - Файл `~/.pgpass` (создайте в домашней директории)

## Подготовка

1. Создайте пользователя, установите пароль и добавьте пользователя в группу sudo, не изменяя его существующее членство в группе
   ```sudo adduser postgres```
   ```sudo passwd postgres```
   ```sudo usermod -aG sudo postgres```
2. Переключитесь на пользователя postgres
   ```su postgres```
3. Создайте Ed25519 ключ, скопируйте ключ на сервер БД и пдключитесь к серверу БД
   ```ssh-keygen -t ed25519```
   ```ssh-copy-id postgres@192.168.1.241```
   ```ssh postgres@192.168.1.241```(в данном случае пользователь ВМ postgres и адрес хоста, где установлена БД, 192.168.1.241)
4. Сделайте проверку подключения к БД
   ```sudo -u postgres psql -c "SELECT version();"``` (в данном случае postgres - это пользователь БД)
5. Вернитесь обратно на хост, где будут храниться бэкапы
   ```exit```
6. Создайте файл в домашней директории пользователя, от которого будет запускаться скрипт:
   ```touch ~/.pgpass```
   ```chmod 600 ~/.pgpass```
7. Запишите настройки подключения
   ```echo "192.168.1.241:5432:*:postgres:password" > ~/.pgpass``` (формат записи `хост:порт:имя_базы:пользователь:пароль`)
8. Создайте директорию для бэкапов
   ```sudo mkdir -p /backups```
9. Настройте права доступа к директории /backups
   ```sudo chown postgres:postgres /backups```
   ```sudo chmod 755 /backups```
10. Создайте директорию для логов бэкапов
   ```touch /var/log/pg_backup.log```
11. Настройте права доступа к логам
   ```sudo chown postgres:postgres /var/log/pg_backup.log```
   ```sudo chmod 644 /var/log/pg_backup.log```

## Установка

1. Скопируйте скрипт в нужное место (`/usr/local/bin/pg_backup.sh`)
   ```sudo cp pg_backup.sh /usr/local/bin/```
2. Добавьте право на исполнение
   ```sudo chmod +x /usr/local/bin/pg_backup.sh```
3. Перейдите в директорию /usr/local/bin/ и запустить
   ```cd /usr/local/bin/```
   ```./pg_backup.sh```

## Настройка запуска скрипта по расписанию

1. Создаqnt файла расписания для пользователя postgres
   ```crontab -e``` (нажмите 1)
2. Внесите запись (запуск скрипта каждую ночь в 2:00)
   ```0 2 * * * /usr/local/bin/pg_backup.sh >> /var/log/pg_backup_cron.log 2>&1```
3. Проверьте, что cron запустился
   ```sudo systemctl status cron```
4. Создайте файл для записи логов cron и установите владельца
   ```sudo touch /var/log/pg_backup_cron.log```
   ```sudo chown postgres:postgres /var/log/pg_backup_cron.log```
5. Проверьте после запуска 
   ```tail -f /var/log/pg_backup_cron.log```
